## Prioritization Criteria

### P1 (High) - Do First
- Blocking other work
- User-facing bug
- Security issue
- Critical integration

### P2 (Medium) - Do Soon
- Important feature
- Performance improvement
- Developer experience
- Technical debt

### P3 (Low) - Do Eventually
- Nice-to-have feature
- Minor improvements
- Exploratory work
- Documentation updates

---

## Effort Sizing

| Size | Description | Typical Duration |
|------|-------------|------------------|
| S | Simple, well-understood | < 2 hours |
| M | Moderate complexity | 2-8 hours |
| L | Complex, may have unknowns | 1-3 days |
| XL | Very complex, research needed | 3+ days |
